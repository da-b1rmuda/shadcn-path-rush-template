# Приоритеты матчингa

Сначала сортируются самые специфичные маршруты:

1. static
2. param
3. optional
4. catch‑all

Глубина маршрута добавляет небольшой бонус. Это гарантирует детерминированный выбор.

## Валидация

Плагин валидации сообщает о:

- дубликатах маршрутов
- неоднозначных маршрутах одинаковой специфичности
- catch‑all не в конце пути
